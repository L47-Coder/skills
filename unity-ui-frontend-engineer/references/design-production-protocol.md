## 1. 读懂 UI 与建立契约

### 确认技术与范围

- 确认本次项目和界面范围、Unity 版本、渲染管线、目标平台，以及使用 uGUI、UI Toolkit、世界空间 UI 或自定义渲染中的哪一种。
- 按用户职责判断 UI，不按目录、Canvas 或组件类型判断。遍历相关 Scene、Prefab/Variant、动态模板和脚本生成入口，把承担信息、操作、状态或反馈的世界空间对象、实体、标签、数字反馈和特效提示纳入。
- 找到权威源头和运行入口：Scene、Prefab/Variant、UXML/USS、模板、生成器、样式配置、Sprite、字体、材质、Shader、Animator/Tween、Addressables/Resources key 和资源加载路径。
- 检查工作树和项目规则，记录用户已有修改及不可覆盖路径。没有证据时不把未知改动归为自己的工作。
- 在任何生产写入前冻结 VCS 状态、Console、编译、测试、适用构建、性能和引用扫描基线；记录命令、Unity 版本、目标平台、Scene、状态、分辨率和结果。

### 观察真实运行

- 运行真实流程并截取现状，覆盖主路径、动态 UI 和关键状态。必要时使用安全的测试入口、模拟存档/数据或 editor-only 驱动。
- 无法运行时继续检查安全入口和引用链，把每项运行行为标为“已验证”或“未验证”，不得凭静态层级猜测。
- 记录真实视口、Canvas Scaler、锚点、安全区、相机关系、排序与遮挡、点击/拖拽区域、输入方式、内容密度、长文本、极端数字、动态生成来源、池化复用和性能预算。

### 分离契约与表现

建立功能契约清单：

- 玩法信息、交互语义、状态和反馈；
- 控制器、数据绑定、输入、焦点和导航；
- serialized field、UnityEvent、脚本/API 和测试入口；
- Prefab Variant override、动画绑定路径、Tween 回调和可中断行为；
- 层级名、组件身份、兄弟顺序、RectTransform、Navigation/Raycaster、UI Toolkit name/class/viewDataKey/query、Localization/TMP fallback；
- GUID、fileID/subasset、managed reference、工厂、池化、动态模板、Addressables/Resources key 和平台约束。

建立表现资产清单：颜色、字体、图标、面板、边框、纹理、背景、布局润色、动效、特效和 UI Sprite。除正式品牌/内容资产或用户明确保留项外，把它们视为待替换占位。字体有授权只证明可用，不自动成为必须保留的风格依据。

### 提炼艺术上下文

- 提炼玩法张力、玩家阅读顺序、单屏停留时长、受众和叙事情绪。
- 从场景、角色、特效和品牌中提取形状、线条、明度、材质、光照、尺度与节奏证据。
- 选取少量高完成度游戏 UI 和跨领域设计作为品质基线，只提炼原则，不复制构图、资产或受保护元素。

在进入概念阶段前，能够用短文和现状截图回答：什么不能坏、什么必须换、UI 为什么要长成这样。

## 2. 主题方向与概念交付

### 方向数量与可比性

- 默认展示 3 个强方向；只有 2 个达到质量线时展示 2 个，不用弱稿凑数，用户可见候选不超过 3 个。
- 对全部候选使用相同真实数据、视口、游戏背景和功能区域，避免用内容差异伪造主题优劣。
- 允许共享已验证的宏观信息架构。只有能解决具体阅读或操作问题时，才把布局变化作为独立 UX 提案。

### 每个方向的完整契约

- 用“玩法张力 → 视觉组织原则 → 玩家情绪结果”写一句主题命题；世界观或装置隐喻只在有项目依据时使用。
- 定义可跨界面复用的 DNA：阅读路径、形状/边缘、排版、图标/线条、材质/光影、动效节奏，并列出 3 个反目标。
- 至少展示一张代表性全屏实景图，以及一条 100% 尺寸的组件/状态细节带。
- 覆盖主交互、信息组件、浮层/反馈等代表性载体，让用户看到完整主题而不是 Hero 图或配色板。
- 说明最主要的体验差异、实现路径和 Unity 回填风险，用图承担主要表达，不用长文掩盖弱视觉。

### 概念工具边界

- 使用生成图、绘制图、HTML 或隔离 Unity 原型提高探索质量。
- 只把生成式位图当作方向选择材料，统一标记 `CONCEPT ONLY` 并排除在生产 Assets、Build 和 Addressables 之外；任何像素、裁片、描摹或衍生纹理都不得直接晋升正式资产。
- 选定方向后，用可编辑原型重建精确层级、交互和组件状态；Unity 真实约束始终是最终依据。
- 不使用简陋几何块、默认字体、临时拼贴、emoji、Unicode 符号、假 AI 文案或系统占位图标冒充高保真概念。

### 展示与推荐

`CONCEPTS` 路径先展示总览选择板和各方向大图。每个方向最多给 3 条短说明，解释主题命题、体验差异和主要风险；明确推荐一个方向及理由，然后进入 Gate A 等待选择。

`DIRECTION_BRIEF` 路径不伪造多个候选，只展示一个足够具体的主题命题、DNA、3 个反目标、约定范围、UX/功能边界和主要实现风险，然后进入 Gate A 请求批准或修订。代理自行具体化不等于用户批准。

## 3. 审美质量门

每个对外候选、最终垂直切片和迁移后的最终界面都必须通过适用质量门；失败项先重做，不交给用户筛弱稿。概念候选按视觉意图与可实现路径评估，垂直切片和最终界面必须用真实 Unity 资产与运行结果验证。

1. **项目特异性**：去掉标题和 Logo 后，仍能从形状、排版、材质和节奏识别当前游戏。无项目依据的霓虹赛博、玻璃拟态、羊皮纸或手游金边套路直接淘汰。
2. **焦点与层级**：在 25% 缩略图、模糊和灰度下仍有一个清晰主焦点、明确阅读路径和不超过两个次焦点；所有面板同权重即失败。
3. **去装饰成立**：移除发光、纹理、网格、角标、假刻度、序列号和主题文案后，组件身份和信息层级仍成立。装饰必须支持分组、状态或反馈。
4. **工艺完整**：在 100% 尺寸检查字体、图标光学重量、描边、圆角、阴影、9-slice、间距和抗锯齿；不使用临时符号或占位图标冒充正式资产。
5. **实景可读**：放回代表性游戏画面后仍清楚、可点、不过度遮挡，长时间观看不过亮、不过吵，不靠常驻发光维持层级。
6. **系统一致**：核心组件能追溯到同一主题 DNA，同时用形态和状态清楚区分语义；统一不等于重复同一块边框。
7. **可生产**：每个独特决定都有 Unity 实现路径、资产计划、性能代价和成本边界；只有离线图好看而无法稳定回填的方向不得通过。

## 4. 深化设计系统与垂直切片

### 收敛选中方向

- 只深化用户选中的主题；融合多个方向时，先让用户确认保留的 DNA 和冲突处理，不做无边界混搭。
- 把颜色角色、字体层级、间距、形状、描边、材质、图标、动效和反馈收敛为 token 与视觉语法。
- 建立组件族、变体和状态，不为每个页面复制孤立样式，也不做布尔参数爆炸的万能 Prefab。
- 使用真实文案、真实数据、目标分辨率和游戏背景设计正常、空、满、极限、加载、错误、禁用、选中、悬停/按下和输入焦点等适用状态。

### 控制 UX 变更

- 把美术方向选择与信息架构、操作路径或大范围布局变更分开。
- 需要 UX 变更时，单独展示问题证据、体验收益、迁移成本、功能风险和回退方案，取得用户确认后再实施。
- 不为了证明主题不同而搬动功能区，也不以视觉统一为理由改变业务语义。

### 完成最高风险垂直切片

- 选择最能暴露布局、动态数据、输入、动效、材质或性能风险的一条真实路径。
- 在目标 Unity 技术栈中接入真实控制器、数据、动态内容、主要状态和主要动效。HTML 或其他工具只辅助迭代。
- 为每个独特效果写清 Sprite/9-slice、字体、材质、Shader、粒子或动画的实现方法和性能代价；无法可靠映射时简化或重做。
- 在真实尺寸、真实背景和 Player/Play Mode 中同时检查视觉、功能、输入和性能。全部通过后才冻结规格并扩展到全部约定范围。

## 5. Unity 回填协议

### 修改前保护

- 再次检查工作树和用户改动，明确本批拥有的文件和对象；无法隔离冲突时转向独立切片或请求用户决策。
- 先做共享 token、基础 Prefab/Variant、材质、字体和 Atlas 的反向依赖与所有权审计；存在范围外消费者时使用 scoped variant/副本，不直接污染共享基线。
- 快照对象路径、组件类型、serialized field、UnityEvent、Prefab Variant override、动画路径、UXML name/class/viewDataKey/binding-path/query、Localization/TMP、工厂/池化约束、资源 key、导入设置和依赖关系。
- 结构性改动使用 Unity Editor 或其序列化 API；不手工改 Unity YAML，不用删除后重建组件/子资产的方式保留表面 GUID。保护 GUID、fileID/subasset、managed reference 和 Variant override，改后重新导入并运行引用扫描。
- 禁止未经确认的 Apply/Revert/Unpack Prefab 操作；变更 base 或共享资产后验证全部下游消费者。
- 不使用破坏性 Git 操作覆盖用户修改，不把 Play Mode 临时变化误写成源资产。

### 沿用真实架构

- uGUI 修改 Prefab、组件和主题配置；UI Toolkit 修改 UXML、USS 和模板；世界空间 UI 修改真实 Prefab、材质与渲染链路。
- 不为局部视觉效果引入另一套 UI 框架。只有结构与行为确实一致时才共享同一 Prefab/模板；仅视觉语言相同不等于结构必须相同。
- 把 token 映射到项目现有的主题配置、颜色/排版常量、ScriptableObject、Prefab Variant、USS 变量、生成器或本地样式工具。
- 把布局映射到 Anchor、Pivot、RectTransform、LayoutGroup、LayoutElement、ContentSizeFitter、Canvas Scaler、Safe Area 和适配规则。
- 把动效映射到 AnimationClip、Animator、Timeline、Tween 或 C# 状态机，保留时长、延迟、缓动、回调、中断与循环语义。

### 分批迁移

- 垂直切片通过后，按“最小 token/原语 → 稳定共享组件 → 界面批次 → 动态与世界空间 UI”迁移，每批完成视觉与功能验证后再继续。
- 把概念视觉重建为可维护的 Sprite、矢量图、9-slice、TMP Font Asset、Atlas、材质、Shader、粒子和动画；不把整屏概念位图当界面实现。
- 修改权威序列化源，不只修改 Play Mode 临时实例、截图或非权威副本。保持 GUID、fileID/subasset、导入设置、资源依赖和可重复生成流程稳定。
- 把动态实例改动落到源 Prefab、模板、生成器、主题配置或创建逻辑，不只修一个运行实例。
- 批量工具只写入已确认的 token、贴图和组件规则，不代替逐屏设计判断。

### 旧资产与授权

- 先让旧素材停止被新系统引用，再做反向引用检查。
- 除非用户明确要求且已证明零引用，否则不自动删除旧 Sprite、字体、材质、Prefab 或主题资源。
- 使用有授权的字体、图标、纹理和素材；不照搬参考作品，不把生成图中的受保护元素直接投入项目。

## 6. 验证与交付证据

### 状态与输入矩阵

按适用范围验证：

- 正常、空、满、长文本、语言扩张、极限数字、加载、失败、禁用、悬停、按下、选中和输入焦点；
- 键鼠、手柄、触屏、焦点导航、点击/拖拽命中、安全区和主要分辨率；
- 动态实例、池化复用、列表滚动/虚拟化、动画中断/循环和异步资源加载。

### 视觉检查

- 用同尺寸 Unity 截图与冻结规格比较首要阅读路径、焦点、对齐、间距、裁切、遮挡、层级、点击区和主题一致性。
- 在代表性游戏背景、真实数据和目标平台上检查，不使用空白编辑器背景掩盖实景问题。
- 检查旧主题颜色、字体层级、面板底图、边框、纹理和装饰是否仍污染选中方向；如果仍像旧主题换皮，回到设计系统修正。

### 功能与资源检查

- 验证无 Missing Script/reference，serialized field、UnityEvent、查询、动画、Variant、Addressables/Resources key 和动态实例仍工作。
- 验证运行态 UI 和脚本生成元素已回填到真实源头，`.meta`/GUID、导入设置和依赖关系稳定。
- 运行相关编译、测试、Player/Addressables 构建、Console 检查和 `git diff --check`。
- 使用与写入前相同的矩阵复验，并为每项记录 `pass`、`fail` 或 `not-run`；区分基线已有问题与本次新增回归。

### 性能检查

- uGUI：检查 Canvas/Layout rebuild、批次、Atlas、Mask 和 overdraw。
- UI Toolkit：检查 style/layout/repaint、动态图集、复杂选择器和列表虚拟化。
- 世界空间 UI：检查相机、排序、Raycaster、像素密度、透明填充率和遮挡。
- 对照项目预算判断；没有预算时记录实测基线与变化，不凭感觉宣称性能可接受。

### 交付证据

提供：

- 用户选择的方向及最终主题 DNA；
- 已覆盖的界面、组件和状态清单；
- 带 Unity 版本、目标平台、commit、Scene、状态和分辨率的真实 Player/Play Mode 截图或录屏；不接受 HTML、Scene View 或概念图替代；
- 功能、构建、Console 和性能验证结果；
- 未验证项、已知限制和后续风险。

无法执行的项目级验证必须明确报告并给出复验步骤。缺少必需运行环境时停在垂直切片或当前安全批次，不开展无法验收的全量迁移。只有概念图、HTML 或静态 Prefab 截图时，不得宣称 Unity 回填已经完成。
